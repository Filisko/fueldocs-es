Introducción

Fuel es un framework dirigido por la comunidad y está abierto a contribuciones por parte de cualquiera que sienta que puede mejorarlo o añadir una característica que llevará el framework a un nivel superior. Ahí son múltiples las formas en que esto se puede lograr. Desde escribir tutoriales, escribiendo y publicando paquetes, hasta contribuyendo a las partes oficiales de Fuel (Núcleo, ORM, Documentación, etc...).

Contribuir al código

Cuando encuentras un bug o ves uno en la lista de problemas el cual puedes resolver o has resolvido, y quieres contribuir al framework. Use GitHub para mandar un pull request. Dependiendo de la versión en la que estás ejecutando la rama a la que estás haciendo pull. Sigue el siguiente conjunto de pasos para cuando solicites un pull:

  • ¿Está la versión que estoy usando actualmente siendo soportada?
    Si no, actualiza a la versión actual, y mira si el bug sigue existiendo.
  • ¿Está ese bug presente en la versión actual?
    Si es así, actualiza a la actual rama en desarrollo, y mira si el bug está arreglado ahí.
  • Si no, comprueba la rama de desarrollo actual en tu repositorio local
    $ git checkout -b 1.8/develop origin/1.8/develop
  • Arregla el bug y hazle commit en tu repositorio local
    Usa un mensaje commit entendible y descriptivo, y asegúrate de que lo firmes.
    $ git commit -m "Esta es mi buena descripción."
  • Dale push a tu cuenta de GitHub
    $ git push
  • Solicita el pull
    Solicita el pull en la misma rama ya que estas (= en la rama actual de desarrollo). Usa la interfaz "update commit range" para dirigir tu petición a la rama correcta. Las pull requests solo son aceptadas en la rama de desarrollo actual. (1.8/develop).
  • Espera para tu momento de fama
    Tan pronto como tu pull request sea aceptado serás parte de la lista de fama de contribuidores.

Puedes encontrar este procedimiento describido con más detalle en Github.

Contribuciones no-bug

Contribuciones no-bug son nuevas características, optimizaciones y otras alteraciones. Todas estas peticiones deben ser hechas en la rama de desarrollo más actual. Así que haz un pull a la rama 1.8/develop, realiza los cambios y haz un pull request.

Para características grandes y/o completamente nuevas, o características que tomarán un tiempo completarlas, podría ser prudente crear una rama separada para la característica. Contacta a uno de nuestros desarrolladores del núcleo vía correo o en IRC para discutir las opciones.
