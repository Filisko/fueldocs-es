Paquete Paraser

El paquete parseador extiende la clase View del núcleo para que puedas usar cualquier tpo de plantilla parser.

Introducción

El paquete Parser no cambia en nada de la forma en que trabajas con las Vistas, pero te permitirá usar cualquier plantilla parser que quieras en vez de PHP. El paquete viene con muchos drivers para parsers como Dwoo, Haml, Jade, Lex, Markdown, Mustache, PHPTal, Smarty y Twig.

El parser que se usa es determinado por la extensión del archivo que usas para tu vista. Las vistas normales, parseadas por PHP, usan la extensión .php. Esta también es la extensión por defecto, si no especificas la extensión se asume que es el archivo de la vista tiene la extensión .php.

Todas las demás tienen su propia extensión, por ejemplo foo.mustache usará el driver the Mustache, foo.tpl usará del driver de Dwoo, foo.lex usará el driver de Lex, y etc. La extensión por defecto puede ser cambiada en la configuración.

Instalación

El paquete parseador es incluído en la descarga de Fuel. Para usarlo debes primero habilitarla añadiendo a tu archivo de configuración:

'always_load' => array(
	'packages' => array(
		'parser',
	),
),

Para que pueda ser posible usar uno de los parsers soportados, tienes que instarlos vía composer. Simplemente añade las librerías a tu proyecto composer.json y entonces ejecuta php composer.phar install:

{
    "require": {
        "dwoo/dwoo" : "*",
        "mustache/mustache" : "*",
        "smarty/smarty" : "*",
        "twig/twig" : "*",
        "mthaml/mthaml": "*",
		"pyrocms/lex": "*"
    }
}

Date cuenta de que el parseador de Markdown es instalado por defecto, ya que también es usado en la clase del núcleo de FuelPHP Markdown.

Librerías que no pueden ser instaladas mediante composer se espera que sean instaladas en APPPATH/vendor/lib_name (capitaliza lib_name), y vas a tener que descargarlos por tu cuenta. No cambies nada, mantenlo tan original como puedas sin el directorio vendor/lib_name para seguir actualizándolo fácilmente (también porque algunos vienen con su propio autoloader).

Configuración

El parseador puede ser configurado en el archivo config/parser.php y admite las siguientes llaves:

Parámetro Tipo Por defecto Descripción
extensions array (matríz con las extensiones y las clases View que las parsean) Define qué extensiones son válidas y cuál es su driver de Vista.
ej: 'md' => 'View_Markdown'

Para cada driver de Vista soportado, el archivo de configuración contendrá una matríz con el mismo nombre que el driver, el cual lista la configuración del driver específico (limitado a los elementos de configuración soportados por el drivers). Esto podría ser un subconjunto de todas las configuraciones que el parser soporta).

Runtime Configuration

Todos los drivers tienen un método parser() que te permitirá acceder al objeto parser actual. Esta es una convención y no un requisito y podría ser diferente en drivers de terceros.

// Limpia el caché para una plantailla Smarty específica.
$view = View::forge('example.smarty');
$view->parser()->clearCache('example.smarty');

// Ejemplo de uso estático
View_Smarty::parser()->clearCache('example.smarty');

Esto también puede ser usado cuando necesites pasar configuración personalizada al driver.
