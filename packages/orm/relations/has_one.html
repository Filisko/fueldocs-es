ORM

ORM es la abreviatura de Object Relational Mapper lo cual hace 2 cosas: mapea las tablas de tu base de datos a objetos y te permite establecer relaciones entre estos objetos.
Sigue el patrón Active Record de cerca, pero también fue influenciado por otros sistemas.

Relaciones: Has One (Tiene un)

Especifica una relación de uno a uno a otro modelo. El modelo elegido debe incluir una referencia "Belongs To (Pertenece a)" al modelo actual para permitir la relación inversa.

Ejemplo

Digamos que tenemos un modelo Model_User y que este modelo has one (tiene un) Model_Profile (que en cambio pertenece a Model_User). El ID de Model_User es guardado con la instancia Model_Profile en su propia tabla. Esto significa que la tabla profiles tendrá una columna llamada user_id (o algo más que tu configures), mientras la tabla usuario no mencionará el perfil. Si te mantienes a las convenciones todo lo que necesitas añadir es añadir la propiedad estática $_has_one a Model_User:

protected static $_has_one = array('profile');

Más abajo hay ejemplos para establecer y acabar con relaciones has-one (tiene un):

// tanto el objeto principal como el relacionado son nuevos:
$user = new Model_User();
$user->profile = new Model_Profile();
$user->save();

// tanto el objeto principal como el relacionado ya existen
$user = Model_User::find(6);
$user->profile = Model_Profile::find(1);
$user->save();

// acabar con la relación establecida mas arriba
$user = Model_User::find(6);
$user->profile = null;
$user->save();

Ejemplo de configuración completa con valores por defecto

// en un Model_User que tiene un perfil
protected static $_has_one = array(
	'profile' => array(
		'key_from' => 'id',
		'model_to' => 'Model_Profile',
		'key_to' => 'user_id',
		'cascade_save' => true,
		'cascade_delete' => false,
	)
);
