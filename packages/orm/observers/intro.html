ORM

ORM es la abreviatura de Object Relational Mapper lo cual hace 2 cosas: mapea las tablas de tu base de datos a objetos y te permite establecer relaciones entre estos objetos.
Sigue el patrón Active Record de cerca, pero también fue influenciado por otros sistemas.

Observadores: Introducción

Un sistema basado en eventos que te permitirá añadir comportamientos a eventos específicos. Cuando un evento es observado el ORM automáticamente llama a todos los observadores añadidos para ver si algo se tiene que hacer.

  • Añadiendo observadores a tu modelo
  • Observadores incluídos
  • Creando tus propios observadores

Añadiendo observadores a tus modelos

Puedes añadir observadores de 2 maneras: simplemente añade el nombre del observador que será llamado en todos los eventos o añade un observador como nombre y una matríz con eventos específicos en los que el observador será llamado.
Cuando el observador está en el mismo espacio de nombre que el modelo y prefijado con Observer_ puedes dejar fuera el prefijo "Observer_". En cualquier otro caso tienes que proporcionar el nombre de la clase completa.

class Model_Article
{
	protected static $_observers = array(
		'example', // llamará el observador Observer_Example en todos los eventos
		'Orm\\Observer_CreatedOn' => array(
			'events' => array('before_insert'), // solo llamará el observador Orm\Observer_CreatedOn en el evento before_insert (antes de insertar)
		)
	);
}
