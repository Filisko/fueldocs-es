Clases

¿Qué es una clase?

Una clase es simplemente una clase normal de PHP. No necesita extender nada o seguir alguna otra convención que no sea la de convención de nombres que es la misma que usan todas las clases de Fuel.

Cargando clases

A diferencia de algunos otros frameworks las clases no necesitan ser cargadas automáticamente. Pueden ser autocargados cuando los referencias en tu código (ya sea desde controladores, modelos, etc).

Clases en subdirectorios

Tal y como las clases de los controladores, las clases deben estar en minúscula exceptuando la primera letra que está en mayúscula, mientras los guiones bajos (_) pondrán a las clases en subdirectorios.

Clases y espacio de nombres

Para determinar qué archivo PHP cargar, el autoloader de FuelPHP tratará los espacios de nombres y los nombres de las clases con guiones bajos de exactamente la misma manera. Esto significa que para las clases dentro de sub-directorios, podrás mezclar y combinar nombres de espacios y guiones bajos para satisfacer ts necesidades.

Toma para ejemplo la clase del archivo app/classes/core/system/messages.php. La clase de este archivo puede ser definida como:

// nombre de espacios global, nombre de clase totalmente subrayado
class Core_System_Messages {}

// combinada un nombre de espacios y guiones bajos
namespace Core;
class System_Messages {}

// o totalmente con espacio de nombres
namespace Core\System;
class Messages {}

El primer método es el primero más común y fácil de entender. Las versiones con nombre de espacios son particularmente manejables si los combinas con la declaración Use en las clases que usan las clases definidas.

Inicializando tu clase

Es posible tener el autolaoder de Fuel realizando tareas automatizadas certeras una vez que la clase sea cargada, algo así como lo que hace __construct() para las instancias de una clase. Esto se hace añadiendo un método public static _init() a la clase.

class Example {
	public static function _init()
	{
		// esto se llamará al cargar la clase
	}
}

Si una clase cargada tiene su propio método _init() y una clase padre debe ser cargada junto a ella, el método init de esta clase padre no será llamado a no ser que lo llames usando parent::_init();.
