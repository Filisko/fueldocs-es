Configurando tu aplicación

Introducción

El framework FuelPHP está construido con configuración por encima de convención en mente. El resultado es una base altamente personalizable en la cual puedes empezar construyendo tu aplicación. The FuelPHP framework is built with configuration over convention in mind. This results in a highly customizable

Los archivos de configuración están guardados dentro de la carpeta app/config. La configuración básica está en app/config/config.php. La configuración puede ser específica según el entorno, más acerca de esto en entornos.

Para sobreescribir ajustes de configuraciones específicas de funciones, o bien puedes crear un archivo nuevo en app/config o puedes copiar la configuración correspondiente desde core/config a app/config.

Opciones de configuración

Estas son las opciones que pueden ser definidas en el archivo de configuración básico, app/config/config.php. Cuando acabas de instalar FuelPHP, este archivo está vacío. Todos los valores por defecto, definidos más bajo, están definidos correspondientemente en la carpeta core/config. Cuando quieras sobreescribir uno por defecto, añade la llave (key) al array en el archivo de configuración de tu aplicación, y modificar el valor. Alternativamente, copia la sección del archivo de configuración del núcleo al archivo de configuración de tu aplicación, y modifícalo.

FuelPHP usa "notación-punto" cuando se manejan matríces. Es una manera conveniente de acceder a elementos de matríces multi-dimensionales. Esta notación es usada en la documentación también, ya que provee una manera fácil de escribir refiriéndose a las matríces. Por ejemplo: 

// cuando ves "always_load.packages = array()", es la forma correcta de:
array("always_load" => array("packages" => array(...) ) );
Llave Tipo Por defecto Descripción
base_url string 
null
 La URL base de la aplicación. Puede ser relativa. DEBE contener una barra diagonal ('/foo/', 'http://ejemplo.com/')
url_suffix string 
''
 Cualquier sufijo que necesita ser añadido a la URL generada por Fuel. Si el sufijo es una extensión, asegúrate de añadir el punto. ('.html')
index_file string 
'index.php'
 El nombre del archivo de arranque (bootstrap) principal. Ponlo en falso o elíminalo si estás usando el módulo de apache mod_rewrite.
controller_prefix string 
'Controller_'
 El prefijo de clases usado para encontrar controladores cuando se mapea la URI al nombre del controlador. Tienes que cambiar esto si quieres tus controlladores con espacios de nombre o en una carpeta distinta de app/classes/controller.
profiling bool 
false
 Habilitar o no el perfilador.
cache_dir string 
APPPATH.'cache/'
 El directorio en el cual guardar los archivos de caché. Este directorio debe tener permisos de escritura.
caching bool 
false
 Habilitar o no el cacheo del buscador de archivos (clase Finder).
cache_lifetime int 
3600
 El tiempo de vida de caché del buscador de archivos en segundos.
ob_callback callback 
null
 Callback dado a ob_start(), establecer a ob_gzhandler para habilitar la codificación gzip en la salida.
errors array Matríz que contiene las llaves de configuración para controlar el comportamiento cuando se han detectado errores:
errors.continue_on array 
array(
)
 En qué errores PHP proceder la ejecución. Vea Manejo de errores.
errors.throttle int 
10
 Cuántos errores mostrar antes de parar de mostrarlos. (previene errores fuera de memoria)
errors.notices bool 
true
 Mostrar o no avisos.
language string 
'en'
 El idioma de la aplicación por defecto, usado por la clase Lang.
locale string 
'en_US'
 Usado para setlocale() requerido en algunas instalaciones PHP, establecerlo a false para evitar que pueda ser modificado.
La sintáxis del ajuste puede variar según el sistema operativo, Ubuntu por ejemplo requiere un sufijo .utf8 (la codificación).
encoding string 
'UTF-8'
 El carácter de codificación por defecto de la aplicación.
server_gmt_offset int 
0
 El margen en segundos de la fecha y hora GMT cuando se usa la función time().
Esto es solo para corregir una mala configuración del tiempo en el servidor. La función time() debería devolver siempre el número de segundos desde 1 de Enero 1970 00:00:00 GMT.
default_timezone string 
'UTC'
 La zona del tiempo del servidor.

Si estableces un valor, asegúrate ABSOLUTAMENTE de que la zona horaria configurada concuerda con la zona horaria establecida en tu servidor. Desde que los cálculos de la zona horaria funcionan convirtiendo a GMT, funciona horriblemente mal cuando no concuerda. Esto causará que el tiempo mostrado en tu aplicación sea incorrecta, y también problemas de cookies y sesiones por el incorrecto cálculo de los tiempos de expiración!

log_threshold int 
Fuel::L_WARNING
 El umbral de registro. Desde qué nivel de mensaje registrar, o un array de niveles específicos que deberían de ser registrados. Lee sobre los posibles valores
log_path string 
APPPATH.'logs/'
 El directorio en el que guardar los registros. Este directorio DEBE tener permisos de escritura.
log_date_format string 
'Y-m-d H:i:s'
 El formato fecha/hora usado en los registros.
 
security array Matríz que contiene las llaves de configuración para controlar la seguridad de tu aplicación.
security.csrf_autoload bool 
false
 Usar o no la autocomprobación del CSRF token. Lee más sobre CSRF.
security.csrf_token_key string 
'fuel_csrf_token'
 Qué llaves $_POST buscar cuando se comprueba el token CSRF.
security.csrf_expiration int 
0
 Establece el tiempo de expiración de la cookie CSRF. Cualquier número mayor que 0 es el número de segundos que tardará en expirar la cookie.
security.token_salt string 
'poner su valor salteado aquí para asegurar que sea más seguro nuestro token'
 Un salteado para asegurarse de generar tokens seguros y no predecibles.
security.allow_x_headers bool 
false
 Permitir que la clase Input use "X" cabeceras cuando estén presentes, como por ejemplo HTTP_X_FORWARDED_FOR o HTTP_X_FORWARDED_PROTO.
security.uri_filter array 
array('htmlentities')
 Qué funciones PHP usar para filtrar la URI.
security.input_filter array 
array()
 Qué funciones PHP filtran las matríces de entrada ($_GET, $_POST and $_COOKIE). Pueden establecerse a xss_clean, esto puede causar un impaco en el rendimiento basado en el tamaño de la entrada.
security.output_filter array 
array()
 Qué funciones PHP filtran las variables a establecer en la vista. Puede ser establecido a xss_clean, esto puede causar un impacto en el rendimiento por el tamaño de las variables.
security.auto_filter_output bool 
true
 Cuándo codificar (htmlentities) automáticamente los datos de la vista.
security.whitelisted_classes array() 
array(
'stdClass',
'Fuel\\Core\\View',
'Fuel\\Core\\Presenter',
'Closure'
)
 Con codificación de salida conmutada en todos los objetos incluidos serán convertidos en cadenas o lanzar excepciones a menos que sean instancias de las clases de esta matriz
 
cookie array Matríz que contiene las llaves de configuración que definen los ajustes globales de las cookies:
cookie.expiration int 
0
 Número de segundos antes de que expire una cookie.
cookie.path string 
'/'
 Restringir la ruta a las que puede acceder la cookie.
cookie.domain string 
null
 Restringir el dominio al que la cookie puede acceder.
cookie.secure bool 
false
 Solo transmitir cookies a través de conexiones seguras.
cookie.http_only bool 
false
 Solo transmitir cookies a través de HTTP, deshabilitando acceso Javascript.
module_paths array 
array()
 Rutas a los directorios de contienen módulos. Usado cuando se añaden módulos sin especificar la localización.
 
always_load array Matríz que conteiene los elementos que el framework tiene que cargar cuando se inicializa:
always_load.packages array 
array()
 Qué paquetes autocargar. Especificar de esta manera: array('package') o array('package' => PKGPATH.'path/to/package'). Para que esto funcione la ruta del paquete debe ser establecida en package_paths.
always_load.modules array 
array()
 Qué módulos autocargar. Especificar de esta manera: array('package'). Los módulos autocargados son accessibles vía URL. Para que esto funcione la ruta del módulo de estar establecida en module_paths.
always_load.classes array 
array()
 Qué clases autocargar e inicializar. Cuando quieras autocargar una clase de un paquete o módulo, ¡asegúrate de que también están autocargados!
always_load.config array 
array()
 Qué archivos de configuración autocargar. Cargar un archivo de configuración en un grupo de esta manera: array('config') o array('archivo' => 'grupo'). Si no quieres cargar el archivo en un grpo, establece el nombre de grupo a null: array('archivo' => null)
always_load.language array 
array()
 Qué archivos de idiomas autocargar. Cargar un archivo de configuración a un grupo de esta manera: array('lang') or array('archivo' => 'grupo'). Si no quieres cargar el archivo en un grupo, establece el nombre de grupo a null: array('archivo' => null).

La interactuación con archivos de configuración y ajustes se realiza usando la clase Config. Los archivos de configuración también pueden ser generados usando Oil.
